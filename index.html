Managing servers can be a tricky business. Most servers have multiple services running on them, with even more configuration files, not to mention their dependencies. You might get it right once, but how much debugging did that take? What step did you do first, and what did you do about that error message again?

Fortunately,

Intecture

can help.

Intecture is built around a library-based API with simple abstractions to help you create powerful, automated configuration scripts.

Better yet, all the abstractions are compatible across platforms, so you don’t have to worry about mixing OSes.

Intecture helps you scale efficiently. With all your configuration centralised, you can have 1 or 1000 servers.

Multi-Lingual

Intecture is capable of binding to almost any programming language there is. It currently supports C, PHP and Rust. More bindings are on their way!

No magic

Managing servers shouldn’t be rocket science. We don’t multi-parse, transform, inject into, DSL-ify, lick or sing to your code. We run your code. Once. You can GDB and xdebug until your heart’s content.

Library-based API

Intecture’s API library gives you the building blocks to manage your servers effectively. It looks and feels just like any other library you’ve used. And it works across platforms so you don’t have to!

Self-contained

Intecture only requires a lightweight, dependency free agent to run on your servers. This means you don’t need to install interpreters and project dependencies onto every server you manage.

Getting Started

1. Write your code

2. Run your code

3. Take a break

API

betweenlines/intecture-api

The API component gives you abstractions (“primitives”) to configure your server.

Auth Server

betweenlines/intecture-auth

The Auth component provides a centralised service for authenticating users and nodes. This means that you don’t have to manage public keys in multiple places.

CLI

betweenlines/intecture-cli

The CLI component allows you to run your code, and manage users and nodes from the command line.

Agent

betweenlines/intecture-agent

The Agent component lives on your managed servers and receives requests from the API to run commands. The Agent contains the real smarts for managing your servers.
